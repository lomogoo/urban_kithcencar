# アーバンネット キッチンカー スケジュール

平日のキッチンカー出店スケジュールを月表示カレンダーで管理する Web アプリです。
Apple のデザイン哲学（SF フォント・余白・なめらかな動き）と **Liquid Glass**
（すりガラス）の表現を採用したシンプルで洗練された UI です。

## 主な機能

- **月表示カレンダー** — 前月／翌月の切り替え、「今日」へジャンプ。カレンダービューは画面内に収まる固定高さ
- **空き状況の色分け**（セル上部の色の線で表示）
  - 🔴 空き（出店者0）
  - 🟡 残り1枠（出店者1）
  - ⬜️ 満員（出店者2・正常）
  - 🟣 要調整（出店者3者以上 → 調整が必要）
  - グレー 休日（土日・祝日・任意指定の休日）
- **リストビュー** — 月内の出店予定を日付順に一覧表示
- **休日指定** — 土日・日本の祝日は自動でグレー表示。任意の日も「休日（出店なし）」に設定可能
- **出店者の登録** — 各日に最大2者をプルダウンから登録（選択するだけで即追加。3者以上も登録可能で「要調整」表示）
- **出店キャンセル** — 日付セルから各出店者を「出店をキャンセル」に設定可能。カレンダー／リスト／出力では取り消し線で表示し、**出店回数・出店料の請求・業務完了報告書の集計からは除外**（空き状況の判定もキャンセル分を除いた人数で行う）
- **出店者の管理** — 新規追加・削除（初期9者を自動投入）。ヘッダー右上のアイコンから
- **出店料管理ビュー** — 月ごとに出店者別の「出店回数」と「出店料」を表示
  - 1回につき **2,000円（税別）**、**4回目以降は無料**（＝当月の上限 **6,000円**）
  - 特定の出店者（例：`Route 227s ` + "`" + `Cafe`）は常時無料
  - **無料募集日**：日付セルから各出店者を「この日の出店料を無料にする」に設定でき、料金計算に反映
- **売上分析ビュー** — 各出店日×各出店者の売上実績を記録し、出店者ごとの今月平均・先月平均・差異を可視化
- **出力機能**（アイコンから、ライブラリは押下時のみ遅延読込）
  - カレンダー／リストを **PDF・画像（PNG）** で出力（色の線は含めず、店名は改行して完全表示）
  - 出店料を **Excel（xlsx）** で出力（A列：出店者名／B列：出店日／C列：その日の出店料（税別）／D列：合計請求額（税別）。出店者ごとに出店回数分の行を並べ、出店者名と合計請求額のセルは縦結合）
  - **業務完了報告書を Word（docx）** で出力（リストビューのアイコンから）。宛名・タイトル・業務時間は固定、件名と業務期間は表示中の月で更新。業務内容は当月の **出店日数**（出店者がいる日数）と **延べ出店台数**、および日付ごとの内訳（`5月1日(金): 店名…`）を自動記載。特記事項は空欄
- **レスポンシブ** — スマホ／PC それぞれに最適化

## 技術構成

- フロントエンド：HTML / CSS / Vanilla JavaScript（ビルド不要の静的サイト）
- バックエンド：[Supabase](https://supabase.com)（PostgreSQL）
- ホスティング：GitHub Pages（GitHub Actions で自動デプロイ）

## セットアップ

### 1. Supabase のテーブルを作成

Supabase ダッシュボード → **SQL Editor** で [`supabase_setup.sql`](./supabase_setup.sql)
の内容をそのまま実行してください。`vendors` / `openings` / `holidays` の3テーブルと、
匿名キー（publishable key）で読み書きできる RLS ポリシー、初期出店者が作成されます。

> 接続情報（プロジェクト URL / publishable key）は `app.js` 冒頭に設定済みです。
> 変更する場合は `SUPABASE_URL` / `SUPABASE_KEY` を編集してください。

### 2. デプロイ（GitHub Pages）

このリポジトリには GitHub Pages 用のワークフロー（`.github/workflows/deploy.yml`）が
含まれています。**一度だけ** 次の設定を行ってください。

1. リポジトリの **Settings → Pages** を開く
2. **Build and deployment → Source** を **「GitHub Actions」** に設定

以降、対象ブランチへ push されるたびに自動でデプロイされます。
公開 URL は Actions の実行結果（`Deploy to GitHub Pages` ジョブ）に表示されます。

### ローカルで動かす

静的サイトなので、任意の HTTP サーバーで配信するだけです。

```bash
python3 -m http.server 8000
# → http://localhost:8000
```

## データ構造

| テーブル   | 役割                                                          |
| ---------- | ------------------------------------------------------------- |
| `vendors`  | 出店者（名前）                                                |
| `openings` | 出店予定（日付 × 出店者）。`fee_free`（無料募集日）・`sales`（売上実績）・`cancelled`（出店キャンセル）を含む |
| `holidays` | 任意指定の休日（出店なしの日）                                |

出店料は `openings` を出店者ごとに日付順で走査して計算します。常時無料の出店者・`fee_free` の回は0円かつ課金回数に含めず、通常回は4回目以降を無料とします（`min(課金回数, 3) × 2,000円`）。`cancelled`（キャンセル）の回は集計から完全に除外します（出店回数にも料金にも含めません）。

> **既存DBへの追記：** `fee_free` / `sales` / `cancelled` カラムは `supabase_setup.sql` に含まれています。
> 既にテーブルがある場合は同ファイル内の `alter table ... add column if not exists` を実行してください（再実行しても安全）。
